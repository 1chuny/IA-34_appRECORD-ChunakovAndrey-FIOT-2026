4. Оптимізувати один із маршрутів API.
```
app.get('/flights', async (req, res) => {
    try {
        // 1. Перевіряємо швидкий кеш Redis
        const cachedFlights = await redisClient.get('flights');
        if (cachedFlights) {
            console.log("✈️ Дані віддано з Redis (Оптимізовано)");
            return res.json(JSON.parse(cachedFlights));
        }

        // 2. Якщо в кеші пусто - читаємо з диска (Повільна операція)
        console.log("📂 Читання даних з файлу JSON");
        const flights = readDB(FLIGHTS_FILE);
        
        // 3. Записуємо результат у кеш на майбутнє
        await redisClient.setEx('flights', 60, JSON.stringify(flights));
        res.json(flights);
    } catch (err) {
        res.status(500).json({ message: 'Server error' });
    }
});
```

5. Провести тестування API.
![Скрін 1](/assets/labs/lab5/5.png)

6. Проаналізувати продуктивність до та після оптимізації.
Аналіз продуктивності показав, що завдяки впровадженню Redis-кешування для ендпоінту /flights, час обробки запитів значно скоротився. Первинне звернення до файлової системи (БД) займає більше часу на зчитування та парсинг JSON. Однак наступні запити, завдяки кешу, обробляються за лічені мілісекунди (середній час відповіді p95 склав менше 10 мс під час навантажувального тестування на 1000 запитів). Кеш успішно інвалідується (скидається) при виконанні POST-запитів на додавання нових рейсів, що гарантує консистентність даних.

## Висновки

Під час виконання лабораторної роботи було успішно розроблено, захищено та оптимізовано REST API для системи бронювання авіаквитків на базі Node.js та Express. У ході роботи було реалізовано багаторівневий захист серверної частини за допомогою модулів helmet та express-rate-limit, налаштовано JWT-автентифікацію для безпечного керування доступом, а також інтегровано in-memory базу даних Redis для кешування маршрутів. Згідно з результатами навантажувального тестування за допомогою утиліти Artillery, впровадження кешу дозволило знизити середній час обробки запитів до лічених мілісекунд та значно підвищити стійкість системи до навантажень. Крім того, програмний комплекс було повністю задокументовано з використанням Swagger та контейнеризовано за допомогою Docker, що у підсумку дозволило створити сучасний, масштабований та готовий до production-розгортання бекенд-застосунок.
